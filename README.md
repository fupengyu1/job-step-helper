## ① 摘要

我做了一个给**心智障碍伙伴**用的网页小工具：把岗位里那种又长又绕的 SOP，拆成**一步一屏的执行卡片**，现场打开照着做就行。  
另外我也加了两块“实用但不花里胡哨”的功能：**沟通卡片**和**任务记录导出**，后面做试点时能把“做没做完、用了多久、求助了几次”记下来，方便复盘。

---

## ② 真实场景与需求

### 目标人群
- 轻中度心智障碍伙伴：多步流程记不住、被打断后容易“断片”、表达需求不容易  
- 支持者：店长/带教员工/就业辅导师/家长等

### 痛点
我拿“便利店补货/清洁/打包”这种岗位当例子：
- **步骤多就容易漏**：做到一半被叫走，回来就不知道从哪接着做  
- **口头交代太抽象**：一句话里塞好几个动作，很难跟上  
- **想表达但说不出口**：比如“我没听懂”“我想休息一下”“这里太吵了”  
- **一变化就更难适应**：临时换顺序/换区域/换工具，压力会明显变大

### 现有做法为什么不够
- 纸质SOP/图片卡：一改流程就要重做，现场也不太好“接着上次那一步”  
- 现场口头带教：很吃店长/老师的时间和状态，忙起来就顾不上  
- 专业就业辅导：资源确实紧张，很多时候没法做到“每天、每次都有人陪着”

## ③ 作品介绍

作品名：**一步一屏｜岗位执行支持器**  
形态：**纯前端网页**

### 核心功能
1) **把岗位流程卡片化执行**  
- 一步就一屏：每张卡只讲一个动作，不让人一上来就被信息淹没  
- 操作很简单：开始 / 上一步 / 下一步 / 完成并记录 / 重置本次  
- 现场被打断也不慌：回来还能继续

2) **沟通卡片**  
- 我把常用表达做成“短句按钮”，分成 **我需要… / 我感觉… / 请你…** 三类  
- 点一下全屏显示，店长/老师一眼能看懂，少吵架少误会

3) **任务记录导出**  
- 自动记：岗位、开始/结束时间、用时、停在哪一步、点了几次沟通卡片  
- 一键导出 CSV，后面做试点/写复盘更省事

4) **离线可用**  
- 第一次打开后会缓存，现场网不稳也能继续用

## ④ 用 SOLO 实现的过程

### 1）先做“能用”的最小闭环
我一开始就给自己定了个原则：**先别追求大而全，先保证现场真的能用。**  
所以我把闭环定成四件事：
1. 选岗位  
2. 一步一屏往下走  
3. 需要时能“说出来”  
4. 做完能“记下来”

等这四件事跑通了，再补离线缓存、可复用模板这些加分项。

### 2）SOLO 在里面主要帮我干了这些活
- 把我脑子里那堆“想做的功能”理成清晰结构  
- 快速生成一个能跑起来的网页 Demo  
- 把 SOP 的长句“翻译”成更容易理解的短句卡片  
- 顺手帮我把“怎么验证”想清楚：做没做完、用了多久、求助几次

### 3）我用得最多的 3 个 Prompt
**Prompt 1：把岗位SOP改写成“短句步骤”**  
> 你是心智障碍就业辅导员。请把“便利店补货流程”改写成最多10步、每步不超过12个字、动词开头、避免同时出现两个动作；并给每步补充一个“可能出错点提醒”。

**Prompt 2：生成沟通卡片**  
> 请按“我需要…/我感觉…/请你…”三类，各生成10张现场可用短句卡片，要求：具体、礼貌、可一键展示；同时给出对应的支持者回应话术。

**Prompt 3：做成可复用模板**  
> 请给出一个岗位SOP的JSON schema：包含步骤数组、每步的短句/计时/检查点/提醒；再给一个“清洁桌面”岗位的示例JSON。

### 4）踩过的坑
- **坑1：写得越详细越好？不，反而更乱**  
  一开始我每步写了很多解释，结果就是——字太多，反而更容易分心。后来我强行收敛：每步只留“一个动作 + 一个提醒”。  
- **坑2：句式不统一，理解成本会飙**  
  有的写“检查…”，有的写“整理一下…”，会很跳。后面统一成：动词开头 + 对象明确 + 一眼能判断“做完没”。  
- **坑3：现场网络不稳定是真的常态**  
  所以加了 Service Worker 做基础缓存，至少第一次加载后能继续用。

## ⑥ 验证方式与下一步

### 我现在怎么验证
我先用“可复现的模拟任务”把流程跑一遍，方便快速迭代：
- 选择岗位：便利店补货  
- 过程：执行到第 N 步，模拟被打断 → 回来继续  
- 记录指标：  
  - 完成时间、总用时  
  - 最后步骤  
  - 沟通卡片点击次数

> 这些数据后续直接搬到真实试点里用也很顺：对比纸质/工具、试点前/后、不同带教方式，都会更好说清楚。

### 下一步我想这么推进
1) 找真实机构/用工方小范围试用  
2) 根据反馈继续打磨：  
   - 卡片文案再短一点、再具体一点  
   - 沟通卡片补齐“现场最常用的那几十句”  
   - 给支持者一份更清晰的回应/带教提示  
3) 把岗位模板慢慢攒起来：便利店/咖啡店/仓库分拣/保洁……能复用就复用

## 附：我为什么觉得它现场能用
这次我刻意把目标定得很朴素：**别做成“看起来很厉害但用不起来”的东西**。  
不用下载 App、不用学半天、现场打开就能跟着做；并且还能把过程记下来——我觉得这才更像是能长期稳定支持的工具。
